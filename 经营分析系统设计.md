# 经营分析系统设计概要

## 1. 系统架构

### 1.1 技术栈
- 前端框架：Next.js 16.1.1
- 后端API：Next.js API Routes
- 数据存储：JSON文件
- UI组件：自定义组件库
- 状态管理：React useState, useEffect, useCallback, useMemo
- 数据缓存：前端内存缓存

### 1.2 目录结构
```
/src
  /app
    /(dashboard)
      /business-analysis
        /orders          # 业务订单页面
          page.tsx
        /revenue-dashboard # 收益看板页面
          page.tsx
        page.tsx         # 经营分析主页面
  /api
    /orders             # 订单API
      route.ts
    /local
      /call-logs        # 本地通话清单API
        route.ts
      /recordings       # 本地录音清单API
        route.ts
  /hooks
    use-auto-refresh.ts # 自动刷新钩子
  /lib
    project-id-name-map.ts # 项目ID到名称的映射
  /config
    project-id-name-map.json # 项目ID到名称的映射配置
```

## 2. 功能模块

### 2.1 经营分析主页面
- 显示经营分析模块的入口
- 提供收益看板和业务订单的导航链接

### 2.2 收益看板
- 时间范围筛选（今天、近7天、近15天、本月）
- 自动刷新功能（每30秒）
- 显示关键指标：
  - 预估收益
  - 成功量
  - 接通量
  - 外呼量
- 各外呼团队收益分析
- 各项目名称收益核算
- 数据缓存和预加载机制

### 2.3 业务订单
- 外呼订单明细展示
- 多条件搜索和筛选：
  - 外呼日期范围
  - 外呼手机号
  - 业务类型
  - 项目名称
  - 外呼团队
- 订单管理功能：
  - 新增订单
  - 编辑订单
  - 删除订单
  - 批量修改订单
- 数据导入/导出：
  - 导入CSV文件
  - 导出CSV文件
  - 下载导入模板
- 自动同步录音清单中客户状态为"成功客户"的数据
- 自动刷新功能（每30秒）

## 3. 业务逻辑

### 3.1 收益看板业务逻辑
1. **数据获取**：
   - 从`/api/orders`获取业务订单数据
   - 从`/api/local/call-logs`获取通话清单数据（用于外呼量）
   - 从`/api/local/recordings`获取录音清单数据（用于接通量）

2. **数据处理**：
   - 计算预估收益（基于订单的差额总和）
   - 计算成功量（基于业务订单的总数量）
   - 计算接通量（基于录音清单的总数量）
   - 计算外呼量（基于通话清单的总数量）
   - 按外呼团队和项目名称分组统计收益

3. **数据缓存**：
   - 实现内存缓存，减少重复API调用
   - 预加载常用时间范围的数据

4. **自动刷新**：
   - 每30秒自动刷新数据
   - 静默刷新，不显示加载指示器

### 3.2 业务订单业务逻辑
1. **数据获取**：
   - 从`/api/orders`获取订单数据
   - 从`/api/teams`获取外呼团队数据
   - 从`/api/recordings/success-customers`获取成功客户数据

2. **数据处理**：
   - 同步录音清单中的成功客户数据到订单
   - 按坐席工号和名称查询所属的外呼团队
   - 批量处理订单数据的导入和导出

3. **订单管理**：
   - 新增订单时检查是否存在重复数据
   - 编辑订单时重新计算优惠金额和差额
   - 批量修改订单时支持全部订单或当前筛选结果

4. **自动刷新**：
   - 每30秒自动同步成功客户数据并刷新订单列表

## 4. API接口

### 4.1 订单API (`/api/orders`)
- **GET**：获取订单数据，支持筛选和分页
- **POST**：添加订单，支持批量添加
- **PUT**：更新订单，支持批量更新
- **DELETE**：删除订单

### 4.2 本地通话清单API (`/api/local/call-logs`)
- **GET**：获取本地通话清单数据，支持时间范围筛选

### 4.3 本地录音清单API (`/api/local/recordings`)
- **GET**：获取本地录音清单数据，支持时间范围筛选和分页

### 4.4 成功客户API (`/api/recordings/success-customers`)
- **GET**：获取录音清单中状态为"成功客户"的数据

### 4.5 团队API (`/api/teams`)
- **GET**：获取外呼团队数据

### 4.6 坐席团队API (`/api/agents/team`)
- **GET**：根据坐席工号和名称查询所属的外呼团队

interface Order {

## 6. 性能优化

### 6.1 前端优化
- **数据缓存**：实现内存缓存，减少重复API调用
- **数据预加载**：预加载常用时间范围的数据
- **并行请求**：使用`Promise.all`并行发起API请求
- **批量操作**：支持批量添加、更新订单
- **防抖处理**：对搜索和筛选操作进行防抖处理

### 6.2 后端优化
- **文件缓存**：实现文件缓存，减少重复文件读取
- **并行文件读取**：使用`Promise.all`并行读取多个日期文件
- **分页处理**：支持服务器端分页，减少数据传输量
- **数据过滤**：在服务器端进行数据过滤，减少数据传输量

## 7. 错误处理

### 7.1 前端错误处理
- **网络错误**：捕获并处理网络请求错误
- **数据验证**：对用户输入进行验证
- **错误提示**：使用toast组件显示错误信息

### 7.2 后端错误处理
- **文件读取错误**：处理文件不存在或读取失败的情况
- **数据解析错误**：处理JSON解析失败的情况
- **API参数错误**：处理参数缺失或格式错误的情况

## 12. 总结

经营分析系统是一个用于管理和分析BPO业务数据的系统，主要包含收益看板和业务订单两个模块。系统使用Next.js框架开发，采用前端内存缓存和后端文件缓存相结合的方式提高性能。系统支持多条件搜索、批量操作、自动刷新等功能，能够满足BPO业务的数据分析和管理需求。

系统的设计考虑了性能优化和错误处理，确保在大数据量下仍能保持良好的响应速度和稳定性。未来可以通过添加数据可视化、用户权限控制等功能进一步增强系统的功能和安全性。
