# 日期筛选功能使用说明

## 🎯 功能概述

在数据分析页面新增了**日期筛选功能**，可以根据日期过滤上传的数据源，只显示指定日期或之前的数据文件。

## 📋 功能特性

### 1. 日期选择器

**位置**：数据分析页面顶部控制栏

**操作**：
1. 点击「选择日期」按钮
2. 在日历中选择目标日期
3. 系统会自动筛选数据源

### 2. 智能日期识别

**识别规则**：
- 从文件名中提取 `YYYYMMDD` 格式的日期
- 例如：`铁通升档20260312.xlsx` → 识别为 2026-03-12
- 对于没有日期的文件名，始终显示

**筛选逻辑**：
- 只显示文件日期 ≤ 选中日期的数据源
- 自动选择第一个符合条件的数据源
- 如果没有符合条件的数据源，显示空状态

### 3. 实时响应

- 选择日期后立即更新数据源列表
- 自动重新执行数据分析
- 保持分析结果与所选日期同步

## 🚀 使用步骤

### 步骤 1: 上传数据文件

1. **访问上传页面**：http://localhost:5000/data-sources/upload
2. **输入数据名称**：建议包含日期，如 `铁通升档20260312`
3. **选择文件**：上传包含客户数据的 Excel/CSV 文件
4. **点击上传**：等待上传完成

### 步骤 2: 使用日期筛选

1. **访问分析页面**：http://localhost:5000/data-sources/analysis
2. **点击日期选择器**：选择目标日期
3. **查看筛选结果**：数据源下拉框会自动更新
4. **执行分析**：点击「一键分析」

### 步骤 3: 查看分析结果

- 系统会使用筛选后的数据源进行分析
- 客户明细、ARPU 值等都会基于所选日期的数据
- 支持导出报告功能

## 🔧 技术实现

### 前端实现

**文件**：`src/app/(dashboard)/data-sources/analysis/page.tsx`

**核心功能**：
- `selectedDate` 状态管理
- `filteredDataSources` 过滤函数
- 日期变化时的自动响应
- 日历组件和日期格式化

**关键代码**：
```typescript
// 日期筛选
const filteredDataSources = dataSources.filter(dataSource => {
  if (!selectedDate) return true
  
  // 尝试从文件名中提取日期
  const fileName = dataSource.filename
  const dateMatch = fileName.match(/(\d{8})/)
  if (dateMatch) {
    const fileDate = new Date(
      dateMatch[1].substring(0, 4),
      parseInt(dateMatch[1].substring(4, 6)) - 1,
      dateMatch[1].substring(6, 8)
    )
    return fileDate <= selectedDate
  }
  return true
})
```

## 🔍 故障排除

### 问题 1: 日期筛选不生效

**可能原因**：
- 文件名中没有 YYYYMMDD 格式的日期
- 日期格式不正确

**解决方案**：
- 重命名文件，包含正确的日期格式
- 例如：`data-20260312.xlsx`

### 问题 2: 数据源列表为空

**可能原因**：
- 没有符合日期条件的数据源
- 所有数据源的日期都晚于所选日期

**解决方案**：
- 选择较晚的日期
- 上传包含正确日期的文件

### 问题 3: 分析结果未更新

**可能原因**：
- 日期变化时的响应逻辑异常
- 数据源切换失败

**解决方案**：
- 刷新页面
- 重新选择日期
- 检查控制台是否有错误
