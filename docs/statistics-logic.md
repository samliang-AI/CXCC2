## 1. 数据看板

### 1.1 接口信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `/api/dashboard/statistics` |
| 请求方式 | GET |
| 筛选参数 | `startDate`（开始日期）、`endDate`（结束日期） |
| 默认时间范围 | 最近7天 |

### 1.2 统计指标

#### 1.2.1 概览数据（overview）

| 指标名称 | 计算逻辑 | 说明 |
|----------|----------|------|
| totalCalls | 各日呼叫量求和 | 统计周期内总外呼量 |
| connectedCalls | 各日接通量求和 | 统计周期内总接通量 |
| successCalls | 各日成功量求和 | 统计周期内总成功量 |
| qualityRate | 质检平均分 | 范围：85.5% ~ 90.5% |
| totalAgents | 各地市坐席人数求和 | 活跃坐席总数 |

#### 1.2.2 趋势数据（trendData）

按日期维度统计每日数据：

| 指标名称 | 计算逻辑 | 说明 |
|----------|----------|------|
| date | 日期 | 格式：MM-DD |
| calls | 日均呼叫量 × 天数系数 × 波动系数 | 当日外呼量 |
| connected | calls × (0.6 ~ 0.8) | 当日接通量 |
| success | connected × (0.5 ~ 0.8) | 当日成功量 |
| rate | success / calls × 100 | 当日成功率(%) |

**日均呼叫量基础值计算**：
```
全省日均呼叫量 = Σ 各地市日均呼叫量
```

地市日均呼叫量基础值：
- 广州（4401）：180
- 深圳（4403）：200
- 佛山（4406）：150
- 东莞（4419）：160
- 其他地市：100 ~ 180（随机）

#### 1.2.3 地市排名（cityRanking）

按成功量降序排列，取前5名：

| 指标名称 | 计算逻辑 |
|----------|----------|
| totalCalls | 日均呼叫量 × 天数 × 波动系数 |
| connectedCalls | totalCalls × (0.6 ~ 0.8) |
| successCalls | connectedCalls × (0.5 ~ 0.8) |
| rate | successCalls / totalCalls × 100 |

#### 1.2.4 坐席排名（agentRanking）

按成功量降序排列，取前5名：

| 指标名称 | 计算逻辑 |
|----------|----------|
| totalCalls | 基于 successCalls 反推 |
| successCalls | 日均成功量(8~20) × 天数 × 波动系数 |
| rate | successCalls / totalCalls × 100 |

#### 1.2.5 地市明细（cityDetails）

展示所有地市的详细统计数据：

| 指标名称 | 计算逻辑 | 说明 |
|----------|----------|------|
| totalCalls | 日均呼叫量 × 天数 × 波动系数 | 累计外呼量 |
| connectedCalls | totalCalls × (0.6 ~ 0.8) | 接通量 |
| successCalls | connectedCalls × (0.5 ~ 0.8) | 成功量 |
| successRate | successCalls / totalCalls × 100 | 成功率(%) |
| avgDailySuccess | successCalls / 天数 | 日均成功量 |
| agentCount | 日均呼叫量 / 15 + 随机值 | 坐席人数 |
| avgSuccessPerAgent | successCalls / agentCount | 人均成交量 |
| expectedRevenue | successCalls × (50 ~ 150) | 预期收入(元) |

#### 1.2.6 质检分布（qualityDistribution）

| 等级 | 数量范围 |
|------|----------|
| 优秀 | 35 ~ 45 |
| 良好 | 40 ~ 50 |
| 合格 | 15 ~ 20 |
| 不合格 | 5 ~ 8 |

### 1.3 数据流程

```
用户选择时间范围 → API请求 → 计算天数
    ↓
生成趋势数据（按日遍历）
    ↓
生成地市排名（按地市遍历，排序取TOP5）
    ↓
生成坐席排名（按坐席遍历，排序取TOP5）
    ↓
生成地市明细（按地市遍历，排序）
    ↓
计算概览数据（汇总计算）
    ↓
返回完整数据
```

## 2. 团队看板

### 2.1 接口信息

| 项目 | 内容 |
|------|------|
| 主接口路径 | `/api/reports/team/statistics` |
| 坐席明细接口 | `/api/reports/team/agent-daily` |
| 请求方式 | GET |
| 筛选参数 | `year`（年份）、`month`（月份）、`teamId`（团队ID） |
| 默认时间范围 | 当前年月 |

### 2.2 统计指标

#### 2.2.1 概览数据（overview）

| 指标名称 | 计算逻辑 | 说明 |
|----------|----------|------|
| totalAgents | 筛选后的坐席数量 | 坐席总人数 |
| totalMonthlySuccess | 各坐席月成功量求和 | 月总成功量 |
| avgCompletionRate | 各坐席完成率求平均 | 平均完成率(%) |
| topPerformers | 完成率 ≥ 100% 的坐席数 | 优秀坐席数 |

#### 2.2.2 团队汇总（teamSummary）

按团队维度汇总：

| 指标名称 | 计算逻辑 |
|----------|----------|
| agentCount | 团队内坐席数量 |
| totalMonthlySuccess | 团队内坐席月成功量求和 |
| totalMonthlyTarget | 团队内坐席月目标求和 |
| avgSuccessPerAgent | totalMonthlySuccess / agentCount |
| completionRate | totalMonthlySuccess / totalMonthlyTarget × 100 |
| topPerformers | 完成率 ≥ 100% 的坐席数 |

#### 2.2.3 坐席明细（agentDetails）

| 指标名称 | 计算逻辑 | 说明 |
|----------|----------|------|
| monthlySuccess | 日均成功量 × 当月天数 × 波动系数 | 月实际成功量 |
| monthlyTarget | 日均成功量 × 22 × (1 ~ 1.2) | 月目标量（按22个工作日计算） |
| completionRate | monthlySuccess / monthlyTarget × 100 | 目标完成率(%) |

**日均成功量基础值**：5 ~ 20（随机）

#### 2.2.4 坐席每日数据（agent-daily）

按日统计单个坐席的数据：

| 指标名称 | 计算逻辑 | 说明 |
|----------|----------|------|
| date | 日期 | 格式：YYYY-MM-DD |
| totalCalls | 基础外呼量 × 周末系数 × 波动系数 | 当日外呼量 |
| connectedCalls | totalCalls × (0.5 ~ 0.8) | 当日接通量 |
| successCalls | connectedCalls × (0.4 ~ 0.7) | 当日成功量 |
| successRate | successCalls / totalCalls × 100 | 当日成功率 |

**计算系数**：
- 周末系数：工作日=1，周末=0.3
- 波动系数：0.7 ~ 1.3（随机）
- 基础外呼量：30 ~ 80（基于坐席工号计算）

### 2.3 数据流程

```
用户选择年月和团队 → API请求
    ↓
计算当月天数
    ↓
根据团队筛选坐席列表
    ↓
生成坐席明细数据（按坐席遍历）
    ↓
生成团队汇总数据（按团队聚合）
    ↓
计算概览数据（汇总计算）
    ↓
返回完整数据
```

## 3. 类型筛选

### 3.1 接口信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `/api/reports/type-filter/statistics` |
| 请求方式 | GET |
| 筛选参数 | `startDate`、`endDate`、`cityCode`、`teamId`、`agentCode`、`keyInfo` |
| 默认时间范围 | 最近7天 |
| 数据来源 | 通话清单 |

### 3.2 统计指标

#### 3.2.1 概览数据（overview）

| 指标名称 | 计算逻辑 |
|----------|----------|
| totalCalls | 各类型总呼叫量求和 |
| connectedCalls | 各类型接通量求和 |
| noAnswerCalls | 各类型未接通量求和 |
| successCalls | 各类型成功量求和 |

#### 3.2.2 类型明细（keyInfoDetails）

按类型分组统计：

| 指标名称 | 计算逻辑 | 说明 |
|----------|----------|------|
| keyInfo | 类型编码 | 如：1, 2, 3... |
| keyName | 类型名称 | 如：类型1, 类型2... |
| teamName | 筛选条件对应的团队名称 | 筛选坐席时显示其所属团队 |
| agentName | 筛选条件对应的坐席名称 | 筛选坐席时显示 |
| totalCalls | COUNT(*) | 总呼叫量 |
| connectedCalls | COUNT(isConnected=true) | 接通量 |
| noAnswerCalls | COUNT(isConnected=false) | 未接通量 |
| connectRate | connectedCalls / totalCalls × 100 | 接通率(%) |
| successCalls | COUNT(isSuccess=true) | 成功量 |
| successRate | successCalls / totalCalls × 100 | 成功率(%) |
| avgCallDuration | AVG(callDuration) WHERE isConnected=true | 平均通话时长(秒) |

#### 3.2.3 展开明细

**按地市分布（cityDetails）**：

| 指标名称 | 计算逻辑 |
|----------|----------|
| cityName | 地市名称 |
| totalCalls | COUNT(*) GROUP BY cityCode |
| connectedCalls | COUNT(isConnected=true) |
| successCalls | COUNT(isSuccess=true) |

**按团队分布（teamDetails）**：

| 指标名称 | 计算逻辑 |
|----------|----------|
| teamName | 团队名称 |
| totalCalls | COUNT(*) GROUP BY teamId |
| connectedCalls | COUNT(isConnected=true) |
| successCalls | COUNT(isSuccess=true) |

### 3.3 数据流程

```
用户设置筛选条件 → API请求
    ↓
从通话清单获取原始记录
    ↓
应用筛选条件（地市/团队/坐席/类型）
    ↓
按类型分组统计
    ↓
计算各项指标（接通率、成功率、平均时长）
    ↓
生成展开明细（地市分布、团队分布）
    ↓
计算概览数据（汇总）
    ↓
返回完整数据
```

### 3.4 筛选逻辑

```
筛选条件优先级：坐席 > 团队 > 地市

筛选坐席时：
- 只返回该坐席的通话记录
- 显示坐席名称和所属团队名称

筛选团队时：
- 返回该团队所有坐席的通话记录
- 只显示团队名称

筛选地市时：
- 返回该地市所有通话记录
- 不显示团队和坐席名称
```

## 4. 外呼结果

### 4.1 接口信息

| 项目 | 内容 |
|------|------|
| 接口路径 | `/api/reports/outbound-result/statistics` |
| 请求方式 | GET |
| 筛选参数 | `startDate`、`endDate`、`cityCode`、`teamId`、`agentCode` |
| 默认时间范围 | 最近7天 |

### 4.2 统计指标

#### 4.2.1 概览数据（overview）

| 指标名称 | 计算逻辑 | 说明 |
|----------|----------|------|
| totalCalls | 各状态数量求和 | 总外呼量 |
| successRate | 成功客户占比 | 成功率(%) |
| failRate | 失败客户占比 | 失败率(%) |
| noAnswerRate | 秒挂无声 + 开场拒访 + 语音助手 占比之和 | 未接通率(%) |

#### 4.2.2 状态明细（statusDetails）

按客户状态分类统计：

| 状态名称 | 基础数量 | 说明 |
|----------|----------|------|
| 成功客户 | ~5 | 已成功办理业务 |
| 高频骚扰 | ~6 | 高频骚扰电话 |
| 开场拒访 | ~634 | 开场白阶段拒访 |
| 秒挂无声 | ~740 | 秒挂或无声 |
| 失败客户 | ~679 | 业务办理失败 |
| 验证码失败 | ~2 | 验证码获取失败 |
| 语音助手 | ~510 | 接通语音助手 |

**数量计算公式**：
```
实际数量 = 基础数量 × 天数系数 × 筛选系数 × 波动系数

天数系数 = 天数 / 7
筛选系数：坐席=0.06，团队=0.12，地市=0.15，全部=1
波动系数 = 0.8 ~ 1.2（随机）
```

| 指标名称 | 计算逻辑 |
|----------|----------|
| count | 实际数量 |
| percentage | count / totalCalls × 100 |
| isHighlight | 是否需要高亮显示（异常状态） |

**高亮状态**：高频骚扰、开场拒访、秒挂无声、失败客户、语音助手

### 4.3 数据流程

```
用户设置筛选条件 → API请求
    ↓
计算天数和筛选系数
    ↓
按状态计算数量和占比
    ↓
计算概览数据
    ↓
返回完整数据
```

## 5. 数据脱敏规则

### 5.1 被叫号码脱敏

所有被叫号码在展示时进行脱敏处理。

**脱敏函数**：`src/lib/utils/mask.ts` - `maskPhone()`

**规则**：
```
显示前3位 + **** + 后4位

示例：
- 13812345678 → 138****5678
- 02012345678 → 020****5678
```

**处理长度**：
- 长度 ≥ 7：正常脱敏
- 长度 < 7：不脱敏，原样显示

### 5.2 应用场景

| 场景 | 是否脱敏 |
|------|----------|
| 通话清单表格 | ✅ |
| 通话清单导出CSV | ✅ |
| 录音清单表格 | ✅ |
| 录音播放面板 | ✅ |
| 质检管理表格 | ✅ |
| AI质检分析 | ✅ |
